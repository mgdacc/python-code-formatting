Cheat Sheet de Python

Code Formatting

1 / 19

¿Por qué es tan Importante?

El código se lee muchas más veces de las que se escribe. Un formato consistente y limpio hace el código más legible, mantenible y fácil de depurar.

2 / 19

El Zen de Python

La filosofía de Python, accesible con `import this`, nos da una pista clave: "Readability counts" (La legibilidad cuenta).

3 / 19

PEP 8: La Guía de Estilo Oficial

Python Enhancement Proposal 8 es el documento que establece las convenciones de estilo para el código Python. Es la "biblia" del formateo en Python.

4 / 19

PEP 8: Indentación

Usa 4 espacios por nivel de indentación. Nunca uses tabuladores.

def mi_funcion():
    if True:
        print("Indentado con 4 espacios")
5 / 19

PEP 8: Longitud de Línea

Limita todas las líneas a un máximo de 79 caracteres (o 99 en algunos proyectos modernos) para mejorar la legibilidad.

6 / 19

PEP 8: Importaciones

Las importaciones deben estar en líneas separadas y agrupadas: 1. Librería estándar, 2. Librerías de terceros, 3. Módulos locales.

import os
import sys

import requests

from mi_proyecto import utils
7 / 19

PEP 8: Espacios en Blanco

Usa espacios alrededor de operadores y después de las comas para que el código "respire".

# Bien
x = 1
mi_lista = [1, 2, 3]

# Mal
x=1
mi_lista=[1,2,3]
8 / 19

PEP 8: Nombres

snake_case para variables y funciones.
CamelCase para clases.
MAYUSCULAS_CON_GUIONES para constantes.

MI_CONSTANTE = 10
mi_variable = 20

def mi_funcion(): pass
class MiClase: pass

El Problema: Formateo Manual

Recordar y aplicar todas estas reglas manualmente es tedioso, propenso a errores y una fuente de debates inútiles en equipos de desarrollo.

10 / 19

La Solución: Autoformateadores

Herramientas que reformatean automáticamente tu código para que cumpla con un estilo predefinido. ¡Dejas que la máquina se encargue!

11 / 19

black: El Formateador Intransigente

Es el formateador más popular. Su lema es: "Cualquier código formateado por Black se ve igual". Acaba con los debates sobre estilo.

12 / 19

black en Acción

Transforma el código desordenado en un formato limpio y estándar.

# Código antes de Black
def mi_funcion( arg1,arg2 ):
    return arg1*arg2

# Código después de ejecutar Black
def mi_funcion(arg1, arg2):
    return arg1 * arg2
13 / 19

Cómo usar black

Se instala con `pip` y se ejecuta desde la terminal en la carpeta de tu proyecto.

pip install black

# Formatea todos los archivos .py en la carpeta actual y subcarpetas
black .
14 / 19

isort: El Ordenador de Imports

Una herramienta especializada que ordena y agrupa automáticamente tus importaciones según las reglas de PEP 8.

pip install isort

# Ordena los imports en todos los archivos
isort .
15 / 19

Linters vs. Formatters

Formatter (`black`): Reescribe tu código para que se vea bien.

Linter (`flake8`, `ruff`): Analiza tu código y te avisa de errores de estilo, bugs potenciales y complejidad.

16 / 19

Integración en el Flujo de Trabajo

Los profesionales integran estas herramientas en sus editores de código (para formatear al guardar) y en sistemas de control de versiones (con `pre-commit`) para garantizar que todo el código subido al repositorio esté formateado.

17 / 19

Regla de Oro

Elige un formateador (casi siempre `black`), úsalo consistentemente en todo tu proyecto y deja de preocuparte por el estilo manual.

18 / 19

Conclusión

El formateo automático del código no es un lujo, es una práctica estándar que ahorra tiempo, mejora la calidad del código y facilita la colaboración en equipo.
